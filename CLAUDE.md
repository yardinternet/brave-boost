# brave-boost

## Guidelines and skills authoring

Keep all content in `.ai/guidelines/` and `.ai/skills/` concise and token-efficient:

- No filler, preamble, or pleasantries
- No restating what the code already says
- Instructions: imperative fragments, not prose
- Examples only when a pattern is non-obvious
- Skip "Introduction", "Overview", "Background" sections
- One rule per line; no padding between related rules
